# 11 — Notifications

Three channels: **in-app**, **email**, **Capacitor push** (FCM).

---

## 1. In-App Notifications

### What it is
A notification badge on the Home tab icon + a notification centre sheet (slide-up) listing recent alerts.

### Trigger events
| Event | Message |
|---|---|
| Task overdue | "You have {n} overdue tasks" |
| Harvest approaching (3 days before) | "{crop} in {plot} is ready to harvest in 3 days" |
| Rain expected today | "Rain expected — consider skipping today's watering" |
| Task completed (confirmation) | "{crop} watered in {plot}" (toast, 2s) |

### Implementation
- Frontend: Angular `NotificationService` — maintains an in-memory list of `AppNotification[]` (signal-based).
- Badge: unread count displayed on `BottomNavBarComponent` settings/home icon.
- Centre: bottom sheet with `TaskListItemComponent`-styled rows. "Mark all read" button.
- Overdue alerts generated by a timer effect on app resume (Capacitor `App.addListener('resume')`).
- Rain alert generated by weather selector (see `10-weather-integration.md`).
- No backend involvement — computed entirely client-side from store state.

## 2. Email Notifications

### What it is
Daily digest email — morning (8 AM) and evening (7 PM) — listing today's pending tasks.

### Provider
**SMTP via `fastapi-mail`** (uses any SMTP provider — Mailgun, Resend, Gmail SMTP, etc.).
Add `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`, `SMTP_FROM` to `.env`.

### Email content

**Morning email (8 AM):**
Subject: "🌱 Your garden tasks for today"
Body:
- Greeting + date
- List of today's tasks grouped by plot
- Weather note if rain expected ("Rain today — no watering needed")
- CTA button → app

**Evening email (7 PM):**
Subject: "🌿 End-of-day garden check"
Body:
- Tasks completed today: N / N
- Outstanding tasks: list
- Tomorrow's tasks preview
- CTA button → app

### Scheduling
APScheduler (`apscheduler` library) running in the FastAPI lifespan:

```python
scheduler = AsyncIOScheduler()
scheduler.add_job(send_morning_reminders, "cron", hour=8, minute=0, timezone="UTC")
scheduler.add_job(send_evening_reminders, "cron", hour=19, minute=0, timezone="UTC")
```

`send_morning_reminders`:
1. Query all users with `morning_reminder=True`
2. For each user: fetch today's tasks → render email template → send via `fastapi-mail`

Email templates: Jinja2 HTML in `backend/app/templates/email/`.

### `.env` additions
```env
SMTP_HOST=smtp.mailgun.org
SMTP_PORT=587
SMTP_USER=postmaster@yourdomain.com
SMTP_PASSWORD=changeme
SMTP_FROM=arboretum@yourdomain.com
```

## 3. Push Notifications (Capacitor + FCM)

### What it is
Native push notifications on iOS and Android via Firebase Cloud Messaging (FCM). Also works as Web Push on Chrome/Edge.

### Setup
1. Create Firebase project → get `google-services.json` (Android) + `GoogleService-Info.plist` (iOS)
2. Add `@capacitor/push-notifications` to frontend
3. On app init: request permission → get FCM token → `POST /api/v1/notifications/push-token` to store in `notification_settings.push_token`

### Frontend (Capacitor)

```typescript
// core/notifications/push.service.ts
async init() {
  const permission = await PushNotifications.requestPermissions();
  if (permission.receive === 'granted') {
    await PushNotifications.register();
  }

  PushNotifications.addListener('registration', (token) => {
    this.store.dispatch(registerPushToken({ token: token.value }));
  });

  PushNotifications.addListener('pushNotificationReceived', (notification) => {
    // Add to in-app notification centre
  });

  PushNotifications.addListener('pushNotificationActionPerformed', (action) => {
    // Navigate based on notification data
  });
}
```

### Backend — FCM send

```python
# backend/app/services/fcm.py
import httpx

FCM_URL = "https://fcm.googleapis.com/fcm/send"

async def send_push(token: str, title: str, body: str, data: dict = {}):
    async with httpx.AsyncClient() as client:
        await client.post(FCM_URL, json={
            "to": token,
            "notification": { "title": title, "body": body },
            "data": data
        }, headers={"Authorization": f"key={settings.FCM_SERVER_KEY}"})
```

### `.env` additions
```env
FCM_SERVER_KEY=your_firebase_server_key
```

### Push triggers
Same schedule as email (8 AM / 7 PM), fired by the same APScheduler jobs:
- If user has `push_token` → send FCM push alongside email
- Payload includes `plot_id` or `task_id` for deep-link navigation on tap

## Notification settings UX

Managed in `07-settings-profile.md`. Three toggles:
- `morning_reminder` — controls both email + push at 8 AM
- `evening_reminder` — controls both email + push at 7 PM
- `in_app_alerts` — controls in-app badge + notification centre

Disabling a toggle: backend skips that user in scheduler queries. FCM token stays stored (re-enabling works immediately).
